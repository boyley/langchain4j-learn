# 企业知识库系统架构说明

## 1. 系统概述

企业知识库是一个基于 AI 的智能问答系统，员工可以通过自然语言查询公司内部文档，系统会根据用户权限返回相关内容。

## 3. 核心流程

### 3.1 文档入库流程

```
原始文档 (.docx/.pdf/.txt)
        │
        ▼
┌───────────────┐
│  文档解析      │  ← 提取文本内容
└───────┬───────┘
        │
        ▼
┌───────────────┐
│  文档分割      │  ← 按段落/句子分割，每块 500-1000 字
└───────┬───────┘
        │
        ▼
┌───────────────┐
│  添加元数据    │  ← department, accessLevel, tags, author, date
└───────┬───────┘
        │
        ▼
┌───────────────┐
│  向量化       │  ← 调用 Embedding API
└───────┬───────┘
        │
        ▼
┌───────────────┐
│  存入向量库    │  ← 向量 + 原文 + 元数据
└───────────────┘
```

### 3.2 查询流程

```
用户提问: "公司的薪资结构是怎样的？"
        │
        ▼
┌───────────────┐
│  身份认证      │  ← 验证用户身份，获取用户ID
└───────┬───────┘
        │
        ▼
┌───────────────┐
│  权限查询      │  ← 查询用户所属部门、可访问部门、最高访问级别
└───────┬───────┘    返回: {部门: 财务部, 可访问: [财务,HR,技术], 级别: confidential}
        │
        ▼
┌───────────────┐
│  构建过滤器    │  ← Filter: department IN [财务,HR,技术]
└───────┬───────┘           AND accessLevel IN [public,internal,confidential]
        │
        ▼
┌───────────────┐
│  问题向量化    │  ← 将问题转为向量
└───────┬───────┘
        │
        ▼
┌───────────────┐
│  向量搜索      │  ← 在向量库中搜索相似文档，应用过滤器
└───────┬───────┘    返回: Top 3 相关文档（已过滤权限）
        │
        ▼
┌───────────────┐
│  生成回答      │  ← 将文档内容 + 问题发给 LLM
└───────┬───────┘
        │
        ▼
┌───────────────┐
│  记录审计日志  │  ← 记录谁看了什么
└───────┬───────┘
        │
        ▼
    返回回答给用户
```

## 4. 权限模型

### 4.1 访问级别

| 级别 | 英文 | 说明 | 可见范围 |
|------|------|------|----------|
| 公开 | public | 所有员工可见 | 全公司 |
| 内部 | internal | 部门内部可见 | 本部门+授权部门 |
| 机密 | confidential | 仅授权人员 | 指定角色 |

### 4.2 权限矩阵

| 角色 | 公开文档 | 本部门内部 | 其他部门内部 | 机密文档 |
|------|----------|------------|--------------|----------|
| 普通员工 | ✓ | ✓ | ✗ | ✗ |
| 部门经理 | ✓ | ✓ | 需授权 | ✗ |
| 总监 | ✓ | ✓ | ✓ | 部分 |
| VP/高管 | ✓ | ✓ | ✓ | ✓ |

## 5. 技术选型

| 组件 | 推荐方案 | 备选方案 |
|------|----------|----------|
| 向量数据库 | Milvus | Pinecone, Elasticsearch, pgvector |
| LLM | GPT-4 / Claude | 文心一言, 通义千问 |
| Embedding | text-embedding-3-small | m3e, bge |
| 文档解析 | Apache Tika | Unstructured |
| 用户认证 | 企业 SSO/LDAP | OAuth 2.0 |

## 6. 部署建议

### 生产环境配置
- 向量数据库：3节点集群
- LLM：调用云服务 API
- 应用服务：K8s 部署，3副本
- 缓存：Redis 集群

### 性能指标
- 文档入库：100 篇/分钟
- 查询响应：< 3 秒（含 LLM 生成）
- 并发支持：100 QPS
