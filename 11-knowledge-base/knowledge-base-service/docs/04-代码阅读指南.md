**目的**: 了解系统有哪些能力

### 文件 3: `source/KnowledgeSource.java`

```java
public interface KnowledgeSource {
    String getName();  // 返回 "database" 或 "file"

    // 全量获取所有文档
    List<KnowledgeDocument> fetchAll();

    // 增量获取 (只获取 lastSyncTime 之后更新的)
    List<KnowledgeDocument> fetchIncremental(LocalDateTime lastSyncTime);

    // 分批获取 (大数据量时用)
    List<KnowledgeDocument> fetchBatch(int offset, int limit);

    // 统计文档数量
    long count();
}
```

**理解要点**:
- 这是知识来源的抽象
- 可以从数据库来，可以从文件来，可以从 API 来
- 只要实现这个接口，就能接入系统

### 文件 4: `store/VectorStoreService.java`

**理解要点**:

## 第三步: 看一个具体实现 (15分钟)

**目的**: 了解接口如何实现

### 文件 5: `source/impl/DatabaseKnowledgeSource.java`

```java
@Component
@ConditionalOnProperty(name = "knowledge.source.database.enabled", havingValue = "true")
public class DatabaseKnowledgeSource implements KnowledgeSource {

    private final KnowledgeDocumentRepository documentRepository;

    @Override
    public String getName() {
        return "database";
    }

    @Override
    public List<KnowledgeDocument> fetchAll() {
        return documentRepository.findByStatus(1);  // 查询所有有效文档
    }

    @Override
    public List<KnowledgeDocument> fetchIncremental(LocalDateTime lastSyncTime) {
        if (lastSyncTime == null) {
            return fetchAll();
        }
        // 只查询更新时间 > 上次同步时间的文档
        return documentRepository.findByStatusAndUpdateTimeAfter(1, lastSyncTime);
    }
}
```

**理解要点**:
- `@ConditionalOnProperty`: 配置 enabled=true 才创建这个 Bean
- `fetchIncremental`: 增量同步的核心逻辑
- 使用 Spring Data JPA 查询数据库

### 文件 6: `store/impl/RedisVectorStoreService.java`

```java
@Service
@ConditionalOnProperty(name = "knowledge.vector-store.type", havingValue = "redis")
public class RedisVectorStoreService implements VectorStoreService {

    private RedisEmbeddingStore store;  // LangChain4j 提供的 Redis 客户端

    @PostConstruct
    public void init() {
        // 启动时初始化 Redis 连接
        this.store = RedisEmbeddingStore.builder()
                .host(host)
                .port(port)
                .indexName(indexName)
                .dimension(dimension)  // 必须是 1536
                .build();
    }

    @Override
    public List<EmbeddingMatch<TextSegment>> search(
            Embedding queryEmbedding, int maxResults, double minScore) {

        // 构建搜索请求
        EmbeddingSearchRequest request = EmbeddingSearchRequest.builder()
                .queryEmbedding(queryEmbedding)
                .maxResults(maxResults)
                .minScore(minScore)
                .build();

        // 执行搜索
        return store.search(request).matches();
    }
}
```

**理解要点**:
- `@PostConstruct`: Spring Bean 创建后自动调用 init()
- `RedisEmbeddingStore`: LangChain4j 封装的 Redis 向量操作
- 搜索逻辑很简单，构建请求 → 调用 search → 返回结果

**目的**: 了解知识是如何被处理的

### 文件 7: `pipeline/KnowledgePipeline.java`

这是最核心的文件，逐段理解:

**同步入口**:
```java
@Transactional
public SyncResult sync(KnowledgeSource source, boolean fullSync) {
    // 1. 获取上次同步时间
    SyncStatus syncStatus = getOrCreateSyncStatus(source.getName());
    LocalDateTime lastSyncTime = fullSync ? null : syncStatus.getLastSyncTime();

    // 2. 从知识源获取文档
    List<KnowledgeDocument> documents = source.fetchIncremental(lastSyncTime);

    // 3. 分批处理
    for (int i = 0; i < documents.size(); i += batchSize) {
        List<KnowledgeDocument> batch = documents.subList(i, end);
        processBatch(batch);
    }

    // 4. 更新同步状态
    updateSyncStatus(syncStatus, "SUCCESS", docCount, null);
}
```

**处理单个文档** (核心):
```java
public int processDocument(KnowledgeDocument doc) {
    // Step 1: 删除旧向量 (防止重复)
    vectorStore.deleteByDocId(doc.getDocId());

    // Step 2: 构建 LangChain4j 的 Document 对象
    Metadata metadata = new Metadata();
    metadata.put("docId", doc.getDocId());
    metadata.put("title", doc.getTitle());
    Document document = Document.from(doc.getContent(), metadata);

    // Step 3: 分割文档成多个片段
    List<TextSegment> segments = getSplitter().split(document);
    // 例如: 1000字的文档 → 分成 3 个 ~350字的片段

    // Step 4: 每个片段向量化并存储
    for (TextSegment segment : segments) {
        // 调用 OpenAI API，把文字转成 1536 维向量
        Embedding embedding = embeddingModel.embed(segment.text()).content();

        // 存入 Redis
        vectorStore.store(embedding, segment);
    }

    // Step 5: 更新数据库状态
    documentRepository.updateVectorStatus(doc.getDocId(), 1, LocalDateTime.now());

    return segments.size();
}
```

**理解要点**:
- 同步 = 获取文档 + 分批处理
- 处理 = 删除旧的 + 分割 + 向量化 + 存储
- 每个文档会产生多个片段，每个片段单独向量化

## 第五步: 看搜索流程 (10分钟)

### 文件 8: `service/KnowledgeSearchService.java`

```java
@Service
@RequiredArgsConstructor
public class KnowledgeSearchService {

    private final EmbeddingModel embeddingModel;
    private final VectorStoreService vectorStore;

    public List<SearchResult> search(String query, int maxResults,
                                      double minScore, String category) {
        // Step 1: 把查询文本转成向量
        Embedding queryEmbedding = embeddingModel.embed(query).content();
        // "如何请假" → [0.15, 0.33, 0.67, ...]

        // Step 2: 在向量库中搜索
        List<EmbeddingMatch<TextSegment>> matches =
            vectorStore.search(queryEmbedding, maxResults, minScore, category);

        // Step 3: 转换成返回结果
        return matches.stream()
            .map(match -> new SearchResult(
                match.embedded().metadata().getString("title"),
                match.embedded().text(),      // 匹配到的文本片段
                match.score(),                // 相似度分数 0-1
                match.embedded().metadata().getString("category")
            ))
            .toList();
    }
}
```

**理解要点**:
- 搜索很简单: 查询文本向量化 → 向量搜索 → 返回结果
- `match.score()` 是相似度，越接近 1 越相似

## 第七步: 看配置 (5分钟)

### 文件 11: `config/EmbeddingModelConfig.java`

```java
@Configuration
public class EmbeddingModelConfig {

    @Value("${knowledge.embedding.openai.api-key}")
    private String apiKey;

    @Value("${knowledge.embedding.openai.base-url:}")
    private String baseUrl;

    @Bean
    public EmbeddingModel embeddingModel() {
        var builder = OpenAiEmbeddingModel.builder()
                .apiKey(apiKey)
                .modelName("text-embedding-3-small");

        if (baseUrl != null && !baseUrl.isEmpty()) {
            builder.baseUrl(baseUrl);
        }

        return builder.build();
    }
}
```

**理解要点**:
- `@Bean`: 告诉 Spring 这个方法返回的对象要被管理
- 其他类通过 `@Autowired` 或构造函数注入就能用 `EmbeddingModel`
